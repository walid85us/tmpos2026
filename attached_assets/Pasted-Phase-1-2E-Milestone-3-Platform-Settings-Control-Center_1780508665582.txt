PART A — CONFIGURATION CENTER OVERVIEW
======================================

Add or improve a top Configuration Center Overview area.

It should summarize:

* total settings
* modified settings
* high-risk settings
* high-risk modified settings
* documentation-only settings
* customized default baseline count
* groups with unsaved changes
* settings by group
* settings by enforcement state

Each card/tile should:

* use the registry/posture data
* be clickable if practical
* apply visible filter/navigation state
* never use invisible filters
* show truthful current-state labels

==================================================
PART B — SETTINGS REGISTRY WORKSPACE
====================================

Add a Settings Registry Workspace view or table.

This should be a scannable governance table over SETTINGS_REGISTRY.

Columns/fields should include:

* key
* label
* group
* owner
* risk
* enforcement state
* current value
* maintained default baseline
* registry default
* modified indicator
* customized default indicator
* impact summary
* truth label
* last changed / audit link if already safely available

This can be a toggle between:

* Grouped View
* Registry View

Or a split layout:

* left navigation / filters
* right registry list

Do not remove the existing grouped editing UX unless fully preserved.
