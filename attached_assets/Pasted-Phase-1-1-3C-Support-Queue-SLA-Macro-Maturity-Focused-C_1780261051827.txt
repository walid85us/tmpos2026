CURRENT QA ISSUES TO FIX
========================

1. Support Queue Center feels like only a filter layer.
   Observed:

* Queue cards click and filter correctly.
* Counts match.
* But the user does not see the operational benefit yet.

Expected:

* Support Queue Center should feel like an operational triage workspace.
* Queue cards should not just behave like filters; they should explain operational meaning and show useful queue intelligence.

2. Queue click-through / active filter is technically working but needs clearer UX.
   Expected:

* Active queue mode should be unmistakable.
* Status tabs / saved views should clearly exit queue mode.
* User should never wonder whether a queue filter is still active.

3. SLA visibility follows permission — FAIL.
   Observed:

* A role without view_support_sla can still see the SLA column under Support Cases.

Expected:

* If view_support_sla is denied:

  * hide SLA columns in case list
  * hide SLA pills in case rows
  * hide SLA indicators in case detail header
  * hide SLA Policy Preview
  * hide SLA-related queue cards or show locked/unavailable state
  * hide SLA fields in Case Operations panel
  * hide SLA workload metrics
* Do not leave partial SLA visibility.

4. Queue memberships on case detail unclear.
   Expected:

* Case detail should clearly show queue membership chips and reasons.
* Example:

  * Unassigned
  * SLA At Risk
  * SLA Breached
  * Escalated Active
  * Pending De-escalation Review
  * Critical / High Severity
  * Recently Updated
  * Stale / No Recent Activity
* If no queue membership applies, show a truthful empty state.

5. Macro placeholder substitution failed.
   Observed:

* Preview shows unresolved text such as:

  * Hi {{tenant_name}}
  * {{case_id}}
    Expected:
* Macro preview should resolve placeholders using the selected support case context.
* Support both token formats:

  * {{tenant_name}}
  * {tenant_name}
* Inserted internal note should also use resolved text, not unresolved raw tokens.
* Missing values should show “Not available” or clearly flagged unresolved placeholders.
* No crash.

6. Missing macro placeholders are not handled clearly.
   Expected:

* Filled placeholders should be visually marked.
* Unavailable placeholders should be marked as “Not available.”
* Preview should not leave raw template syntax unless intentionally marked as unresolved.

7. Macro Management is unclear/missing.
   Expected:
   Implement a lightweight internal Macro Management panel/drawer now.

It should be gated by manage_support_macros and support:

* view macro library
* create macro
* edit macro
* disable/enable macro
* category
* purpose
* body
* placeholder chips
* internal-only truth label
* no external sending
* audit macro create/edit/disable/enable
* safe local/session mock persistence if current app uses mock/session data

If manage_support_macros is denied:

* hide management actions or show read-only library only.

8. Bulk Triage is deferred but not clear.
   Expected:
   Do not implement full bulk triage in this correction unless already safely present.

Instead:

* Add a visible “Bulk Triage — Future” card/panel in Support Tools.
* Explain why it is deferred:

  * requires stronger confirmation, audit, permissions, and batch safety
  * no external automation in this phase
* Mention future scope:

  * bulk assign
  * bulk severity change
  * bulk internal note
  * bulk macro insert
* This panel should be informational only and must not mutate data.
* Document it in replit.md and history file.

9. Command Center and Support Tools still feel somewhat simple.
   Expected for this correction:

* Do not rebuild Command Center.
* Improve Support Queue Center with richer operational value:

  * triage summary
  * top queue reason
  * oldest case
  * SLA pressure
  * owner/team split
  * recommended next action
  * quick “Open queue” action
  * queue explanation
  * maybe a small preview of top cases
* Keep changes focused inside Support Tools.
