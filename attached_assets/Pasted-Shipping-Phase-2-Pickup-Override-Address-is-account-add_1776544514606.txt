==================================================
APPROVED PRODUCT DECISION
==================================================

This is now the approved model:

1) After label purchase, shipment.originAddress must remain locked.
2) Recovery for pickup booking must NOT require changing the printed label address.
3) Add a separate pickup-only dispatch address / pickup-only contact override that is used for the carrier pickup driver, without changing the label from_address.
4) EasyPost createPickup must default is_account_address to false.
5) Add a per-store EasyPost setting for the rare case where the origin address is truly registered with USPS as a shipper-of-record, and only then allow is_account_address=true.
6) Add pickup location detail support (suite / dock / door code / building / hours) for pickup routing.

==================================================
NON-NEGOTIABLE REQUIRED FIXES
==================================================

1) LOCK SHIPMENT ORIGIN AFTER LABEL PURCHASE
Required:
- once label exists / is purchased, shipment.originAddress must no longer be editable
- destination must remain locked as well
- this closes the current latent bug surfaced by the analysis

2) ADD PICKUP OVERRIDE ADDRESS MODEL
Required:
- add optional pickupOverrideAddress on shipment
- add optional pickupOverrideContact if needed
- add pickupLocationDetail for suite/dock/door/building/hours
- resolvePickupAddress must use pickupOverrideAddress when present, otherwise originAddress
- clearly label this in UI as:
  "Pickup dispatch address (used only for the carrier pickup driver; does not change the printed label)"

3) ADD PICKUP OVERRIDE EDITOR
Required:
- add an in-context editor in the pickup section
- prefill from current originAddress
- editable after label purchase
- separate from shipment origin editor
- contact name/phone and pickup location detail should be editable here too

4) PICKUP VERIFICATION / ELIGIBILITY MUST USE THE OVERRIDE WHEN PRESENT
Required:
- pickup verification
- pickup eligibility memory
- pickup create payload
must all use the resolved pickup address source (override first, otherwise origin)

5) FIX EASYPOST is_account_address DEFAULT
Required:
- default is_account_address to false in EasyPost createPickup
- do not silently default it to true anymore
- add a per-store/provider setting:
  "Origin address is registered with USPS as a shipper-of-record"
  default OFF
- only send true when explicitly enabled

6) PICKUP LOCATION DETAIL HANDLING
Required:
- support pickup-specific detail like suite / dock / door / building / hours
- wire it into the pickup payload appropriately:
  - address.street2 where appropriate
  - pickup instructions where appropriate
- keep this provider-aware but honest

7) PICKUP RECOVERY UX
Required:
- when pickup fails for an address-related reason after label purchase, the recovery action must point to:
  - Edit pickup dispatch address
  - Edit pickup contact
  - Edit pickup location detail
  - Re-verify
- do not tell the operator to edit the locked shipment origin
