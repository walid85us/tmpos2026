PART A — SERVICE POINT SEARCH HONESTY

1) REMOVE FAKE ZIP SEARCH BEHAVIOR
Required:
- eliminate the current fake behavior where typing a ZIP just mutates/relabels the same mock list
- do not present preview/mock results as real carrier-authoritative service points

2) HONEST LIVE-LOCATOR-UNAVAILABLE STATE
Required:

  "Live carrier service-point lookup requires carrier-specific production credentials and a carrier-specific locator adapter. Currently unavailable."
- keep this wording honest, concise, and operator-friendly
- do not show active search UI that implies real live lookup if the app cannot actually perform it

3) MANUAL SERVICE-POINT ENTRY FALLBACK
Required:
- operators must still have a usable fallback path
- allow manual service-point details entry / manual service-point reference handling where appropriate
- keep it lightweight and auditable
- do not create a dead-end when live lookup is unavailable

4) ELIGIBILITY-AWARE SEARCH / LOOKUP UX
Required:
- service-point lookup/search UI must only be active when the shipment is otherwise eligible:
  - plan allows it
  - permission allows it
  - provider mode is active
  - provider is configured
  - shipment is PACKED
  - mutual exclusivity allows it
  - origin and destination validation/business rules are satisfied
  - carrier/service context exists if needed
- if any prerequisite is missing, the UI must honestly explain why lookup is unavailable
