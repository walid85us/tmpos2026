PART A — SPLIT-PANE DOMAIN CONTROL PANEL
========================================

Transform DomainsPage into a more professional split-pane control panel.

Recommended layout:

* top command header remains
* posture cards remain
* filters remain
* main area becomes:

  * left pane: Domain Portfolio
  * right pane: Domain Control Panel for selected domain

Left pane:

* grouped domain portfolio
* root/main domains shown as parent records
* related subdomains nested under their root domain
* platform subdomains grouped clearly if applicable
* each domain row/card should show:

  * hostname
  * role badge: Root/Main Domain or Subdomain
  * tenant
  * lifecycle/readiness badge
  * SSL badge
  * risk badge
  * concise next action
* clicking a row selects the domain and opens/updates the right control panel
* keep existing list/filter logic and no-drift counts

Right pane:

* persistent selected domain control panel
* if no domain selected, show strong empty state:
  “Select a domain to manage DNS readiness, SSL readiness, relationships, checklist, and history.”

On small screens:

* it may collapse to current list → drawer behavior if easier
* do not break responsive behavior

==================================================
PART B — ROOT DOMAIN CONTROL PANEL VISIBILITY
=============================================

Root/main domains must be first-class.

When selected domain is a root/main domain, the right panel must clearly show:

Section title:
“Root Domain Management”

Show:

* root hostname
* tenant
* lifecycle status
* DNS readiness
* SSL readiness
* next recommended action
* risk reasons
* related subdomains count
* related subdomain list
* manual action checklist
* registrar/DNS provider external note
* history/audit summary if already available

Related subdomains:

* show child hostname
* tenant if different/applicable
* readiness/SSL/status
* click child to select it

If no related subdomains:

* show:
  “No subdomains attached to this root domain yet.”

==================================================
PART C — SUBDOMAIN CONTROL PANEL VISIBILITY
===========================================

When selected domain is a subdomain, the right panel must clearly show:

Section title:
“Subdomain Management”

* DNS readiness
* SSL readiness
* next recommended action
* risk reasons
* manual action checklist
* history/audit summary if already available

Required explanation:
“Subdomain hostname is generated from an editable label plus a locked root domain.”

==================================================
PART D — PORTFOLIO GROUPING
===========================

Implement root/subdomain grouping.

Requirements:

* root/main domains are visually grouped
* subdomains appear under their parent root when possible
* orphan/legacy subdomains that cannot resolve a root should appear in an “Unlinked / Legacy Subdomains” group with a truthful note
* platform/internal subdomains should appear in a “Platform Domains” or equivalent group if applicable
* grouping should not break filters

When filters are active:

* filtered results should remain clear
* if a root is hidden but a child matches, still show enough parent context so the child relationship is understandable
