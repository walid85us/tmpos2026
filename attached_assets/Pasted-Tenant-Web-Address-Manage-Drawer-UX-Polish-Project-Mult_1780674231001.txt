==================================================
GOAL
====

When the user clicks Manage from the Tenant Web Address table, open a clean, professional right-side drawer/workspace titled:

Tenant Web Address Workspace

The drawer should feel simple, modern, and operational.

It should help the system owner quickly manage:

* the tenant platform web address
* customer-facing access links
* external website / redirect guidance
* setup progress
* recent activity

==================================================
DRAWER STRUCTURE
================

1. Drawer Header

Show:

* Tenant Web Address Workspace
* tenant name
* platform web address
* status chip: Active / Needs Setup / Disabled / Reserved
* close button

Primary actions:

* Copy Link
* Open Site, if safe/active

Use compact, polished buttons.

2. Customer Access Points

Show a clean list/card section for:

* Main App
* Customer Portal
* Book Appointment
* Track Repair
* Pay Invoice
* Mail-In Repair

Each row should show:

* link name
* short URL/path preview
* status:

  * Active
  * Future / Not active
  * Disabled by plan
  * Not configured
* copy action
* open action only if safe

Do not use toggle switches unless enable/disable behavior already exists.

3. External Presence

Show:

* external website URL if recorded
* empty state if none recorded
* redirect target: platform web address
* guidance text:
  “Tenant-owned websites and domains are configured outside this app. For now, redirect the tenant’s website to the platform web address.”
* custom domain support label:
  “Future / Support-Assisted”

Do not show DNS setup, registrar setup, SSL verification, DNSSEC, domain lock, or transfer lock.

4. Configuration Progress

Show a simple progress/checklist section:

* Tenant slug assigned
* Platform web address generated
* Customer portal available
* Booking link available
* Tracking/payment links available
* External redirect guidance reviewed, if applicable

Use truthful statuses only.
Do not imply automatic external verification.

5. Recent Activity

Show a concise activity section using existing audit/history where available.

Examples:

* Platform web address generated
* External website recorded
* Status changed
* Tenant web address disabled/re-enabled

Do not audit passive copy/open actions.
Do not say immutable record.

==================================================
VISUAL REQUIREMENTS
===================

The drawer should be:

* right-side slide-over
* clean and professional
* easy to scan
* no horizontal scrolling
* no nested scroll problems except normal drawer body scroll if content exceeds height
* compact but readable
* not overloaded
* visually closer to a modern SaaS workspace than a technical domain panel

Keep the main table minimal.
Do not add Customer Links or External Website columns back into the table.
