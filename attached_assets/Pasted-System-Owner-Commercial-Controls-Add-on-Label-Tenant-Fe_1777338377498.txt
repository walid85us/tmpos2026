==================================================

- End Trial

5) Feature currently granted as paid override
Display:
- Paid Override or Overridden
- tenant-specific override price if applicable
Actions:
- Edit Price
- Revoke Override

6) Feature disabled by System Owner for that tenant
Display:
- Disabled by Owner
Actions:
- Re-enable if permitted

7) Feature linked to disabled or archived add-on
Display:
- do not show as available add-on
- do not allow new trial/paid add-on grant from inactive catalog item
- if historical assignment exists, show only as inactive/revoked/historical if such view exists

PART D — ADD-ON AVAILABILITY RULE
==================================================

Required:
A feature should be considered “Add-on available” only when:
- it has linkedFeatureId / capability key matching an add-on catalog item
- add-on catalog item is Active
- tenant plan does not already include it, or it is commercially offered separately as an add-on by design

If add-on is Disabled or Archived:
- no Add-on grant button
- no Add-on available label as active option
- linked Store Permissions Matrix permissions should not appear through that add-on entitlement
