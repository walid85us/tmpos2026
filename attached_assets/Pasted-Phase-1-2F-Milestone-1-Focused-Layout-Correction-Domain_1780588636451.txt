==================================================
PORTFOLIO TABLE LAYOUT REQUIREMENTS
===================================

Fix the table so it fits as a professional dashboard/table layout.

Preferred approach:

1. Remove nested vertical scrolling from the table container.
   The portfolio table should expand naturally in the page. The browser/page can scroll if the list is long.

2. Remove horizontal scrolling on normal desktop widths.
   Use a better responsive table design:

* reduce excessive min-widths
* allow controlled column widths
* wrap long domain names cleanly
* use two-line cells where needed
* use compact badges
* combine related secondary text inside cells where useful
* ensure Actions column remains visible

3. Keep all required columns visible on desktop:

* Domain
* Domain Type
* Tenant
* Registrar
* DNS Readiness
* SSL Readiness
* Email DNS
* Security Readiness
* Renewal / Expiry
* Risk
* Next Action
* Actions

4. If the table is still too dense, use professional cell compaction instead of horizontal scroll:

* Domain cell can show hostname on first line and metadata on second line.
* Renewal / Expiry can be compact.
* Next Action can wrap.
* Badges can be smaller but readable.

5. Preserve a useful responsive fallback:

* If a smaller screen truly cannot fit all columns, use a card/list layout or controlled responsive hiding.
* Do not introduce a bad permanent horizontal scrollbar on desktop.
