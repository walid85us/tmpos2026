UI REQUIREMENTS
===============

1. Platform Team Directory Enhancements

Enhance the existing platform team directory/table/cards.

For each platform team member, display as available:

* name
* email
* role label
* status
* last activity or fallback text such as “Not recorded”
* governance category
* risk posture
* recommended review cadence
* access review inclusion recommendation
* system-protected badge where applicable
* future temporary-elevation eligibility label
* access-level summary

Use clear SaaS language.

Example labels:

* System Protected
* Included in Future Access Review
* Temporary Elevation Eligible — Future Workflow
* Advisory Only
* UI-Gated Today
* Server Enforcement Future

If data is missing, use truthful fallback text, not fake data.

Examples:

* “Last activity not recorded”
* “Review workflow not active yet”
* “Temporary access workflow future/deferred”

2. Role Summary / Governance Cards

Add a concise role summary section if it improves clarity.

For each current active role:

* System Owner
* Support Admin
* Billing Admin
* Operations Admin
* Security Admin

Show:

* role label
* purpose
* governance category
* risk posture
* review cadence recommendation
* system protection status
* temporary elevation eligibility
* notes/warnings

Do not introduce active roles such as Read-only Auditor or Platform Admin.

Future role concepts may be shown only in a clearly separated “Future / Deferred Role Concepts” area, if appropriate.

3. Role Matrix UI Preservation

Preserve the existing Global Permissions Matrix.

You may improve readability with:

* clearer section headings
* sensitive sub-permission badges
* advisory enforcement tier labels
* helper text explaining thresholds
* tooltip/copy explaining UI/client-gated status

But do not change the underlying permission levels, thresholds, dependency auto-sync, or resolver logic.

4. Effective Access Preview Preservation

Preserve the existing Effective Access Preview.

You may improve clarity with:

* governance summary labels
* source explanation
* denied prerequisite explanation
* sensitive access indicators
* advisory-only labels

But do not alter the resolver output or access decision behavior.

5. Truthful Banner / Notice

Add a clear notice in the Team Management / Platform Team Governance area:

“Current platform access controls are UI/client-gated in this phase. Server-side enforcement, automated access review, and temporary access workflows are future/deferred.”

Use professional wording.

The notice should not be alarming, but it must be truthful.

6. Empty / Missing Data States

Handle missing data safely:

* missing last activity
* unknown role
* missing governance catalog entry
* missing review status
* missing access summary

Use graceful fallbacks and do not crash.

7. Modern SaaS UI Quality

Use a modern, clean SaaS admin-console layout consistent with the existing app.

Prefer:

* compact badges
* clear cards
* readable tables
* section headers
* short explanatory helper text
* no clutter
* no fake automation claims

Preserve existing visual style.
