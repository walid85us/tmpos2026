==================================================
TABLE POLISH REQUIREMENTS
=========================

1. Actions column must always be fully visible.

* Do not allow the Manage button to be clipped.
* If needed, rename button to a shorter label such as “Open” or use a compact icon + “Open”.
* Keep the action accessible and clear.

2. Registrar column should be compact.
   Current issue: “Amazon Route 53 — platform apex (repairplatform.com)” wraps into too many lines.

Preferred rendering:

* First line: registrar name
* Second line: short muted context only if needed
  Examples:

  * Amazon Route 53
    platform apex
  * Cloudflare
    root DNS
  * GoDaddy
    external registrar

Avoid showing long parenthetical text like “(repairplatform.com)” if it makes the row too tall. Move detailed context to tooltip or future detail view if needed.

3. Renewal / Expiry should be compact.
   Preferred:

* If no date: show “—”
* Add small muted/future pill: “Auto-renew future”
* Do not stack multiple lines unnecessarily.
* If date exists, show date first, then small muted “Auto-renew future” below only if it remains readable.

4. Next Action should wrap cleanly but not make rows excessive.

* Keep it concise.
* Examples:

  * No action needed.
  * Complete manual SSL readiness.
  * Add missing email records.
  * Review security posture.

5. Badges should be compact and consistent.

* Keep current truthful badge language.
* Reduce padding/width if needed.
* Avoid tall badge stacks.

6. Row height should be professional.

* Allow two-line cells where necessary.
* Avoid five-line cells.
* Use tooltips or detail view for long explanatory text.

7. Keep required columns available:

* Domain
* Domain Type
* Tenant
* Registrar
* DNS
* SSL
* Email DNS
* Security
* Renewal / Expiry
* Risk
* Next Action
* Actions

If space is tight, prioritize compact rendering instead of hiding required columns.
