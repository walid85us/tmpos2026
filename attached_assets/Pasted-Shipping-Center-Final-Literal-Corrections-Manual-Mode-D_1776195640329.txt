NON-NEGOTIABLE MODE RULES
==================================================

There must now be two explicit operational shipment modes.

A) Provider Mode
This is active only when the shipment is actually using provider-backed workflow.
Provider Mode characteristics:
- provider is configured and active
- shipment is not in manual carrier/service override mode
- provider address validation is used
- Get Rates is available when prerequisites are met
- provider rate selection is used
- Purchase Label is available according to prerequisites
- provider tracking/provider events govern later lifecycle progression

B) Manual Mode
This is active when Carrier and Service Level are entered manually instead of using a provider-selected rate.
Manual Mode characteristics:
- Provider & Operations must be disabled or clearly inactive
- provider-backed actions must not be usable:
  - validate address
  - get rates
  - purchase label
  - sync tracking
- shipment continues in manual handling flow like before provider integration
- store updates shipment manually according to manual workflow rules
- UI must clearly state that provider is active globally, but this shipment is in manual mode because carrier/service were entered manually
- clearing the manual carrier/service values should allow the shipment to return to provider mode when prerequisites are met

This mode split must be automatic and rule-driven.
It must not rely on a separate user switch.
