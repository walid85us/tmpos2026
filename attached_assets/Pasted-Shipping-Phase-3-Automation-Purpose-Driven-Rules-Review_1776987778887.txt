==================================================
PART A — REPLACE TECHNICAL RULE TYPES WITH BUSINESS PURPOSE
==================================================

Required:
Remove “Observational” and “Guardrail” as primary user-facing choices.

Replace with:

1) Flag / Note
Purpose:
- after an event happens, flag a shipment, add an internal note, or create a visible follow-up marker.

2) Queue for Batch
Purpose:
- mark eligible shipments as candidates for batch labels.

3) Require Review
Purpose:
- require an operator review before a defined next step proceeds, or flag a shipment for review after an event depending on trigger.

4) Require Approval
Purpose:
- require authorized approval before a risky action proceeds.

5) Block Action
Purpose:
- block a risky action until the condition changes or an authorized override is applied, if override is allowed.

Required:
The system may still use internal rule modes if useful, but the operator-facing experience must be purpose-driven.

PART C — SELECTED-RATE REVIEW BEFORE LABEL PURCHASE
==================================================

This is mandatory.

Required workflow:
For rules like:
- selected shipping rate > $6
- require review before label purchase

The correct process is:
1) User selects a rate
2) App evaluates relevant “A carrier label is about to be purchased” rules
3) If selected rate is compliant, allow Purchase Label
4) If selected rate violates a Require Review / Require Approval / Block Action rule:
   - do not purchase the label yet
   - show the reason
   - create a visible pending review/approval/block state
   - allow the user to choose a lower rate if available
   - allow an authorized user to approve/override when permitted
   - after approval/override, allow label purchase
5) If the user selects a lower compliant rate, clear or bypass the pending issue appropriately with audit trail

Required:
The rule must block the next risky step, not the whole shipment.
The user should still be able to change rate or cancel.
