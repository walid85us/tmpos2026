==================================================
PART A — ADD-ON LIFECYCLE STATES
==================================================

Required:
Add-on Catalog lifecycle should support:

- draft
- active
- disabled
- archived

Meaning:

1) Draft
- newly created but not yet commercially available
- visible to System Owner in Add-on Catalog
- represented in Plans & Features Matrix
- not shown as available in tenant Features tab
- cannot be granted as Trial or Paid Override

2) Active
- commercially available
- may appear in Tenant Features tab only if compatible with tenant’s current plan and not included by plan
- can be granted as Trial or Paid Override

3) Disabled
- temporarily unavailable
- not shown as available tenant add-on
- historical records preserved

4) Archived
- retired from active commercial offering
- not shown as available tenant add-on
- historical records preserved
- should only be allowed when no active plan inclusion / compatible-plan dependencies remain

==================================================
PART D — ARCHIVE PROTECTION RULE
==================================================

Required:
Add-on cannot be archived if any of the following are true:

1) The linked feature is enabled in any plan in Plans & Features Matrix.
2) The add-on has any Compatible Plans selected.
3) There are active tenant trials linked to the add-on.
4) There are active paid overrides linked to the add-on.
5) There are pending payment invoices linked to the add-on.
6) There are unresolved active tenant entitlements linked to the add-on.

If any dependency exists:
- block Archive
- show in-app explanation
- do not use browser-native confirm
- list what must be cleared first

Example message:
“This add-on cannot be archived while it is included in Growth plan or offered to Advanced tenants. Remove plan inclusion / compatible plans and resolve active tenant grants before archiving.”

==================================================
PART E — HOW TO ARCHIVE PROPERLY
==================================================

Required:
To archive an add-on, System Owner must first:

1) Remove included-by-plan access from Plans & Features Matrix where applicable.
2) Remove optional plan compatibility from Add-on Compatible Plans.
3) Resolve or revoke active tenant grants.
4) Cancel or resolve pending payment invoices.
5) Then archive.

Important:
Plans included by Plans & Features Matrix are controlled only from Plans & Features Matrix.
Compatible Plans editor should remain locked for included-by-plan plans.
