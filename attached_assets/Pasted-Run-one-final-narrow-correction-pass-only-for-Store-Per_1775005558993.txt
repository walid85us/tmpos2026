Administrative Actions must behave as true sub-permissions under their related Module Access.

and any other related admin actions

If one of those sub-permissions is Denied, the user must NOT be able to do that action even if they still have general module access.

Required fixes:

1) Re-architect Administrative Actions as true sub-permissions of Module Access
Current issue:
- Module Access and Administrative Permissions are shown separately
- but they are not behaving like a real parent/child relationship
- Store Owner cannot reliably use sub-permissions to narrow what a role can do inside an allowed module

Fix:
- make Administrative Actions true sub-permissions tied to their related Module Access domain
- Module Access grants entry to the area/surface
- Administrative Actions further restrict or allow sensitive actions inside that module
- a user may have module access but still be denied specific sub-actions
- this must be true everywhere in the app, not just in the permission page UI

2) Define and enforce parent/child permission rules
Implement these rules globally:

A. Parent rule
- If Module Access is None:
  all related Administrative Actions must be effectively unavailable/inactive/denied
  user cannot access the module or any sub-actions

B. View-only rule
- If Module Access is View only:
  user may view the module where intended
  but cannot perform create/edit/manage/approve sub-actions unless explicitly and intentionally supported
  in most cases, related admin sub-actions should be unavailable/inactive

C. Higher module access rule
- If Module Access is Create / Edit / Manage / Approve / Full Access:
  user may enter the module at that level
  but each Administrative Action must still be separately enforceable as Granted or Denied
- Denied sub-action must block the action even if parent module access is high

D. No implicit bypass rule
- Parent module access must not silently override denied sub-permissions
- A high module level does not automatically grant every sub-action
- Store Owner control over sub-actions must be real and enforceable
