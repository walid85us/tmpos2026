OBJECTIVE
=========

Simplify the Tenant Web Address table so it feels like a clean SaaS management list.

The table should support fast scanning:

* Which tenant?
* What is their platform web address?
* Is it active?
* Do they have an external redirect/custom website note?
* Are customer links available?
* What is the next action?
* How do I open/manage/copy?

Detailed URLs should live inside the selected overview/drawer, not as separate table columns.

TABLE SIMPLIFICATION REQUIREMENTS
=================================

Replace the current detailed table columns with a simpler list.

Preferred table columns:

1. Tenant

* tenant name
* optional small secondary store/plan text if already available

2. Platform Web Address

* tenant platform subdomain
* compact copy action
* optional open action only if safe
* no long extra URL list in this cell

3. Status

* Active
* Needs Setup
* Disabled
* Reserved
* External Only if applicable

4. External Website / Redirect

* show external URL/domain if recorded
* otherwise show “—”
* if custom domain support is future, show small “Future / Support-Assisted” label only where relevant

5. Customer Links

* show a compact summary, not individual URL columns
  Examples:

  * “5 links available”
  * small chips: Portal, Book, Track, Pay, Mail-In
  * disabled/future chips only if needed
* Do not show full URLs here.
* Detailed link list belongs in the selected overview.

6. Next Action

* concise
  Examples:

  * No action needed.
  * Complete setup.
  * Review external redirect.
  * Copy portal link.
  * Custom domain support future.

7. Actions

* Open Overview
* Copy Main URL
* optional More menu if already consistent
* action buttons must not be clipped

Remove separate columns for:

* Main App URL
* Customer Portal URL
* Booking URL
* Tracking URL
* Payment URL
* Mail-In URL

Those details must remain available in the selected overview/drawer.
