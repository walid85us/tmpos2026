==================================================
NON-NEGOTIABLE REQUIRED FIXES
==================================================

1) EDIT DATE / WINDOW MUST ENTER A REAL EDIT STATE
Required:
- clicking Edit Date / Window must visibly and unmistakably move the operator into a usable pickup-date/window edit state
- it is not enough to only scroll or pulse
- the date field must become obviously active/editable
- if needed, also expand or visually isolate the pickup date/window area
- the operator must be able to tell immediately what to change

2) RETRY BOOKING MUST BE VISIBLY REAL
Required:
- clicking Retry Booking must:
  - clear the old failed state
  - show an obvious retry-in-progress state
  - and then show a new result
- if the retry is blocked because the date/window was not changed, say so explicitly
- do not leave the operator unsure whether anything happened

3) FAILED/ORPHAN CANCELLATION MUST HAVE ONE CLEAR MODEL
Required:
- in failed/orphan pickup state, there must be one clear cancellation action path
- do not leave the operator with two actions that appear to mean similar things
- if the bottom generic Cancel Pickup button still exists in that state, fix that
- use one coherent label and one coherent action path for orphan cleanup

4) IMPROVE ACTION FEEDBACK
Required:
- every recovery action must give visible feedback:
  - edit action enters edit state
  - retry action shows progress/result
  - cancel action opens confirm/result
- no dead-feeling buttons

5) DO NOT REOPEN UNRELATED ISSUES
Required:
- do not broaden into UPS DAP policy work
- do not broaden into FedEx future-date policy work
- do not change truthful provider messaging already achieved
- keep this pass narrowly focused on recovery-action functionality
