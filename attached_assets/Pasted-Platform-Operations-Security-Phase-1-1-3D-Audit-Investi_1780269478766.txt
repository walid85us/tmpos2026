PRIMARY GOAL
============

Transform Audit & Security from an audit table into an investigation workspace.

The System Owner / Audit Admin / Security Admin should quickly understand:

* What happened?
* Who did it?
* Which tenant/entity was affected?
* Was this normal, suspicious, high-risk, or restricted?
* What related events happened before and after?
* Has this actor done similar actions?
* Has this tenant/entity had repeated risk events?
* Is there an existing support case linked to this event?
* Should a support case be created from this event?
* Has the event been reviewed?
* What notes/evidence exist?
* Can the investigation be summarized/exported safely?
* What data is current app/session data vs future real SIEM/security tooling?

Use deterministic, transparent, rule-based investigation logic.

Do not implement real AI/ML.
Do not claim SIEM-level detection.
Do not claim immutable audit storage.
Do not claim real threat detection.
Use terms like:

* Rule-based finding
* Investigation signal
* Current app/session audit data
* Related event
* Confidence
* Source
* Reviewed
* Needs review
* Restricted detail
