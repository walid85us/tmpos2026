CURRENT QA-BLOCKING ISSUE
==================================================

The Add-on Catalog, Plans & Features Matrix, and Tenant Features tab are not fully aligned.

Example:
- Add-on Catalog: Loyalty Program
- Linked feature: Loyalty Management
- Add-on Compatible Plans may show only Advanced

This is confusing unless the app clearly treats Plans & Features Matrix as the source of truth for included-by-plan access.

The correct architecture is:

Plans & Features Matrix = source of truth for included-by-plan access.
Add-on Catalog = source of truth for optional commercial add-ons.
Add-on Compatible Plans = controls optional add-on availability only when the feature is NOT already included in the plan.
Tenant Features tab = resolved tenant-facing commercial entitlement result.
Store Permissions Matrix = role permission control after entitlement exists.

==================================================
CRITICAL SOURCE-OF-TRUTH RULE — ADD-ONS MUST EXIST IN PLANS & FEATURES MATRIX
==================================================

All add-ons must be represented in the System Owner Plans & Features Matrix using their linked feature/capability ID.

The Plans & Features Matrix is the source of truth for whether an add-on-linked capability is included in a plan.

This means:
- every Add-on Catalog item must be linked to a feature/capability
- that linked feature/capability must exist in the Plans & Features Matrix
- System Owner can enable that feature directly for Essential / Growth / Advanced from the Plans & Features Matrix
- if enabled in a plan, tenants on that plan receive it as Included by Plan
- if not enabled in a plan, the Add-on Compatible Plans setting controls whether it may be offered as an optional add-on to tenants on that plan

If an add-on is created and its linked feature/capability does not exist in Plans & Features Matrix:
- create or register the feature/capability there
- do not create duplicate feature rows
- preserve the same feature/capability key across Plans & Features, Add-on Catalog, Tenant Features, and Store Permissions Matrix

Tenant Features behavior:
1) Plan includes the add-on-linked feature:
   - show Included by Plan
   - show Add-on
   - do not show Trial / Paid Override for that same entitlement

2) Plan does not include the add-on-linked feature, but tenant’s plan is in Add-on Compatible Plans:
   - show Not in Plan
   - show Add-on
   - show Trial
   - show Paid Override

3) Plan does not include the add-on-linked feature, and tenant’s plan is not in Add-on Compatible Plans:
   - do not show the add-on-derived row as an add-on
   - do not show Add-on
   - do not show Add-on Available
   - do not show Plan Upgrade Required

Compatible Plans editor rule:
- plans where the feature is enabled in Plans & Features Matrix must be locked/read-only in the Add-on Catalog Compatible Plans editor
- System Owner must change included-by-plan access from Plans & Features Matrix, not from Add-on Catalog
- only plans where the feature is not included can be toggled as optional add-on compatible plans

PART D — COMPATIBLE PLANS EDITING BEHAVIOR
==================================================

When System Owner edits Compatible Plans for an add-on:

1) Included-by-plan plans:
- remain locked/read-only
- cannot be unchecked from Add-on Catalog
- must be changed from Plans & Features Matrix

2) Non-included plans:
- can be checked/unchecked as optional add-on-compatible plans

3) When tenant’s plan is added to Compatible Plans:
- add-on-derived row appears in Tenant Features tab
- row shows Not in Plan + Add-on
- row shows Trial + Paid Override actions

4) When tenant’s plan is removed from Compatible Plans:
- add-on-derived row disappears as an add-on from Tenant Features tab
- no Add-on label remains
- no Add-on Available label remains
- no Plan Upgrade Required message appears

This must work consistently across Essential, Growth, and Advanced.
