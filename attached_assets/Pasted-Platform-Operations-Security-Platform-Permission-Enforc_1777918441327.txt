CRITICAL ARCHITECTURE RULE
==================================================

Global Permissions Matrix must control:

1) Sidebar navigation visibility
2) route/page access
3) page-level feature access
4) action buttons
5) sensitive workflow actions
6) sub-permission actions

No hardcoded platform role should override a Matrix denial.

The final access decision must come from:
- active Dev Session platform role
- Global Permissions Matrix parent permission
- Global Permissions Matrix sub-permission
- System Owner permanence rule

Support Tools must not use only the old hardcoded `can()` matrix.
The old `can()` helper may be reused only if it is fed by / reconciled with the Global Permissions Matrix result.

Rule:
If Global Permissions Matrix says a sub-permission is None, the action must not be available for that role.
