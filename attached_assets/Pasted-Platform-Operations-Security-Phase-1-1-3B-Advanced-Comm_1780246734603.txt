==================================

Before answering, implementing, correcting, or fixing anything, think step by step.

Do not rush to code.

1. Inspect the current Command Center implementation.
2. Inspect platformOpsDerive.ts and existing shared derivation helpers.
3. Inspect Support Tools, Audit & Security, Domains, Tenant 360, Add-on Governance, and existing mock/session data sources.

7. Design the Command Center intelligence model.
8. Implement slowly with reusable helpers.
9. Verify no accepted workstream regresses.

PRIMARY GOAL
============

Transform Command Center from a good dashboard into a more advanced operational intelligence center.

The System Owner should quickly understand:

* What needs attention now?
* What changed since last review?
* What is getting worse?
* Which tenant is most at risk?
* Which risk signals are driving the score?
* Which issues are correlated?
* Which cases/escalations/audit events/domain/commercial items are related?
* What should be reviewed next?
* What is noise vs. real priority?
* Which signals are verified vs. inferred?
* What is rule-based now vs. future AI later?

Use deterministic, transparent, rule-based intelligence.

Do not implement real AI/ML in this phase.
Do not label anything as AI-generated.
Use terms like:

* Rule-based insight
* Derived signal
* Confidence
* Explanation
* Source
* Last updated
* Current app/session state

PART E — CORRELATED SIGNALS / OPERATIONS EPISODES
=================================================

Add a rule-based “Correlated Signals” or “Operational Episodes” section.

This should group related signals when they share:

* same tenant
* same case
* same domain
* same severity
* same audit actor
* same time window
* same operational category

Examples:

* Tenant has active escalation + overdue SLA + recent critical audit event.
* Domain failed + support case open for same tenant.
* Commercial blocker + tenant support case.
* Repeated high-risk audit events for same actor/tenant.
* Escalated support case + unassigned owner.

Each group should show:

* episode title
* related signals count
* severity
* tenant/entity
* why grouped
* recommended next action
* drilldown

Truth label:
“Rule-based correlation; not AI prediction.”

Do not create fake incident records.
This is a Command Center intelligence grouping only.
