IMPORTANT SCOPE DECISION — SKIP ADDRESS AUTOCOMPLETE FOR NOW
==================================================

Do NOT implement Google autocomplete, Geoapify autocomplete, OpenStreetMap autocomplete, or any other address-autocomplete feature in this pass.

We are intentionally skipping destination address autocomplete for now.

Instead, the shipping provider’s existing address validation flow must be the required gate before the user can proceed to the next operational step.

This means:
- no more work on autocomplete in this pass
- remove any half-finished or misleading autocomplete UI if needed
- keep the destination address entry manual for now
- make provider-based address validation mandatory before rates
