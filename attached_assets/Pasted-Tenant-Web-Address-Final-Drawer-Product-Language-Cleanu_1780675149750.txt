==================================================
ITEMS TO FIX
============

1. Status vocabulary

Current issue:
The drawer still shows domain-style status such as “Verified.”

Replace user-facing status language with Tenant Web Address language:

Preferred:

* Active
* Needs Setup
* Disabled
* Reserved
* External Only
* Future / Support-Assisted where relevant

Mapping guidance:

* verified → Active
* pending / verifying → Needs Setup
* failed → Needs Review or Needs Setup
* disabled → Disabled

Do not show “Verified” as the primary user-facing status.

2. Manage Web Address controls

Current issue:
The controls still show old domain lifecycle language such as:

* Mark Pending
* Verified

Change visible wording:

* Status: Active / Needs Setup / Disabled
* Mark Active
* Mark Needs Setup
* Disable

Do not show:

* Mark Pending
* Mark Verified
* Verify
* Trigger verification

Internal values may remain unchanged if needed for safe compatibility, but the visible UX should use Tenant Web Address vocabulary.

3. Recent Activity wording

Current issue:
Recent Activity may show old labels such as:

* Subdomain Created

Replace display labels where safe:

* Subdomain Created → Platform Web Address Created
* Domain Created → Web Address Created
* Domain Status Changed → Web Address Status Changed
* Domain Disabled → Web Address Disabled
* Domain Re-enabled → Web Address Re-enabled
* External domain recorded → External Website Recorded

Do not change audit storage or event IDs.
This is a display-label mapping only.

4. Metadata wording

Current issue:
Footer/metadata may show:

* Verified
* Last checked

Replace with softer, truthful wording:

* Created
* Status reviewed
* Last reviewed

Do not use “Last checked” if there is no live DNS/provider check.
Do not imply live verification.

5. Advanced Custom Domain Support panel

The architect flagged this as the only remaining place that may still contain DNS/SSL terminology.

Make it stricter:

Preferred label:
Advanced Custom Domain Support
Future / Support-Assisted

Preferred copy:
“Custom domain hosting is not self-serve in this phase. Tenant-owned websites and domains remain external. Support-assisted setup may be considered in a future phase.”

Avoid default display of technical DNS/SSL details.

If the panel currently contains DNS/SSL/registrar details, either:

* hide those details behind a collapsed future-only disclosure, or
* replace them with the simplified support-assisted language above.

Do not show:

* DNS Zone Editor
* SSL automation
* certificate provisioning
* DNSSEC
* domain lock
* transfer lock
* registrar control
* provider API

6. Configuration Progress visual polish

Current issue:
Completed checklist items appear visually crossed out.

Completed items should use:

* green check icon
* normal readable text
* subtle completed styling

Do not use strikethrough text for completed checklist items because it looks like deleted/invalid text.
