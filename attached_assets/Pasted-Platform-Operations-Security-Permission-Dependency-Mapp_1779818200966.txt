CURRENT QA NOTES TO FIX

1) NBA linked permission issue

Observed:
- view_next_best_actions / NBA was set to View Only.
- act_on_nba_recommendations still had Manage.
- User could still click Open Case and navigate to Support Tools.
- This is logically inconsistent.

Expected:
For linked permission pairs:
- view permission controls section visibility.
- action permission controls actions within that section.
- action permission should not be higher than its required view/prerequisite permission unless explicitly designed and shown.
- If a view permission is downgraded to View Only, related action permission should be capped/downgraded or the UI should warn and auto-reconcile.
- If view permission is None, related action permission should also be disabled/downgraded or made ineffective.
- Action should never route to fallback/overview if target permission is missing.

Preferred implementation:
Add a permission dependency/prerequisite map.

Example:
view_next_best_actions is prerequisite for:
- act_on_nba_recommendations

If view_next_best_actions is None:
- act_on_nba_recommendations must be None or ineffective.
- NBA section hidden.

If view_next_best_actions is View Only:
- act_on_nba_recommendations should not remain Manage silently.
- Either auto-downgrade it to View Only/None or show a dependency warning and make action unavailable.

For this project, implement auto-reconciliation:
- When a prerequisite permission is lowered, dependent child permissions must be capped to the prerequisite level or set to None if action requires mutation.
- Log/audit the dependent permission adjustment.
