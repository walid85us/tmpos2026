When naming helpers, labels, statuses, seed data, and future UI copy, preserve these final wording rules:

   “No live DNS lookup in this phase.”

Do not use:
“Live DNS lookup limitations may apply.”

2. Use:
   “Record Manual Verification”
   or:
   “Mark Records Reviewed.”

Do not use:
“Verify Now”
or:
“Trigger verification.”

3. Use:
   “SSL automation is future. Current SSL readiness is manually tracked.”

Do not imply active certificate issuance.

4. Use:
   “Auto-renew: Future / Not active.”

5. Use:
   “Manual propagation confirmation.”

Do not use live propagation, global propagation %, latency, or nameserver telemetry unless clearly future-labeled.

6. Use:
   “Future telemetry — not active in this phase.”

Do not show real-looking traffic, geolocation, request counts, or analytics as current functionality.

7. Use:
   “Governance Mode”
   or:
   “Policy Baseline — Not Runtime Enforced.”

Do not use “Strict Enforcement” as a runtime claim.

8. Use:
   “Review / Discard / Save through group review.”

Do not use “Approve / Reject” unless a future approval workflow is explicitly labeled.

9. Use:
   “Audit record.”

Do not use “immutable record” unless immutable storage is actually implemented.

10. Use:
    “Operator manually marked verified.”

Do not use:
“System automatically verified.”

OBJECT MODEL TO ADD OR EXTEND
=============================

Create or extend local-only domain models safely.

Do not destructively migrate existing records.

Preferred approach:
Keep existing TenantDomainRecord and tenant_domains_v1 as the base domain source of truth.

Add sibling local/session stores as needed:

* tenant_dns_records_v1
* tenant_domain_registrar_v1
* tenant_domain_security_v1
* tenant_domain_ssl_view_v1 if needed
* tenant_domain_email_dns_v1 if needed

Only add separate stores where helpful.
Do not over-engineer if a derived helper is enough.

Required concept 1 — DnsRecord

Fields should support:

* id
* domainId
* type: A | AAAA | CNAME | TXT | MX | SPF | DKIM | DMARC | CAA | NS
* host / name
* value / target
* ttl
* priority optional
* source:

  * platform_required
  * operator
  * email_required
  * security_required
  * registrar_required
* status:

  * intended
  * manual_ready
  * manually_verified
  * conflict
  * pending_manual_review
  * not_applicable
* purpose
* lastEditedAt
* notes optional

Important:
This is an intended-state zone mirror, not a live DNS zone.

Required concept 2 — DomainRegistrarInfo

Fields should support:

* domainId
* registrarName
* ownershipNote
* nameservers optional
* expiryAt placeholder
* autoRenew: unknown | enabled | disabled | future
* transferLock: unknown | enabled | disabled | future
* domainLock: unknown | enabled | disabled | future
* dnssec: unknown | enabled | disabled | future
* providerNotes optional

Important:
Registrar/security fields are local readiness/future placeholders only.

Required concept 3 — EmailDnsReadiness

Derived from DnsRecord records:

* mx
* spf
* dkim
* dmarc
* overall
* nextAction
* missingRecords
* requiredRecords
* truthLabel

Status vocabulary should be manual/readiness based:

* ready
* partial
* missing
* pending_manual_review
* not_applicable

Required concept 4 — SslCertificateView

Derived/local:

* status

* issueReason

* requiredValidationRecord optional
