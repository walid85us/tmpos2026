PART A — PERMISSION DEPENDENCY AUTO-SYNC MODEL
==================================================

Implement direction-aware dependency reconciliation.

Do NOT implement unsafe full bidirectional mirroring.

Use this professional IAM rule:

1) Raising a dependent action permission should auto-raise prerequisites to the minimum needed view level.
Example:
- If act_on_nba_recommendations is changed to Edit / Approve / Manage / Full Access:
  - view_next_best_actions must be auto-raised to at least View Only if it was None.
  - do not auto-raise it beyond View Only unless required.

2) Lowering a prerequisite permission should cap or downgrade dependent action permissions.
Example:
- If view_next_best_actions is changed to View Only:
  - act_on_nba_recommendations should be reduced to View Only or None because it is a mutation/action permission.
- If view_next_best_actions is changed to None:
  - act_on_nba_recommendations should be set to None or marked ineffective.
Preferred:
  - auto-set dependent mutation permissions to None.
  - audit the auto-adjustment.

3) Lowering a dependent action permission should NOT lower prerequisite view permission.
Example:
- If act_on_nba_recommendations is set to None:
  - view_next_best_actions may remain View Only because read-only visibility is still valid.

4) Explicit System Owner Full Access remains locked/permanent.

5) All auto-reconciliations must be auditable.
