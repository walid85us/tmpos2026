PART A — AUDIT & SECURITY SURFACE
==================================================

Build a platform-wide Audit & Security surface.

Required sections:

1) Security Posture Cards
Show truthful status cards such as:
- Production log redaction: Enabled
- Add-on archive/delete protection: Enabled
- Tenant feature entitlement source of truth: Plans & Features Matrix
- Store Permissions Matrix protection: Enabled
- Shipping credential safety: Configured / Not configured / Unknown
- Pending high-risk items: count
- Failed/blocked actions: count
- Recent security-relevant events: count

Important:
Do not fake security controls.
If a control is not actually implemented, show:
- Not Implemented
- Unknown
- Manual Review Required
instead of pretending it exists.

2) Platform Audit Viewer
Create a filterable audit/event viewer using existing audit sources where available.

Fields should include:
- timestamp
- actor
- tenant
- action
- entity type
- entity id/name
- severity
- category
- result/status
- note/reason

Filters:
- date range
- tenant
- actor
- category
- severity
- action text search
- entity type

Categories should support at minimum:
- Commercial Controls
- Add-on Governance
- Billing / Invoice
- Tenant Management
- Security / Access
- Shipping
- Support
- Domain
- Platform Settings
- Team Management

If some categories do not yet have events, show empty state truthfully.

3) Audit Detail Drawer / Modal
Clicking an audit row should open a detail view with:
- full safe metadata
- old/new value summary if available
- related tenant link
- related invoice/add-on/feature/support/domain link if available
- no PII beyond what is already allowed by current app context

4) Event Severity
Support severity labels:
- info
- notice
- warning
- critical

Do not invent critical events if none exist.

5) Security Notes
Allow System Owner to add an internal security note to an audit event if the data model supports it.
If not practical, keep notes at the Support/Security issue level.
