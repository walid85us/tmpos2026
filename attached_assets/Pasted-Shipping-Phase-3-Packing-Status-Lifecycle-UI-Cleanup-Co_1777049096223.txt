PART B — GET RATES ENABLEMENT AFTER PACKED
==================================================

Required:
Get Rates should remain blocked until packing readiness is satisfied.

Get Rates should be enabled only when:
- origin address is validated
- destination address is validated
- packages exist
- package data is complete and verified as needed
- shipment is Packed because packing was completed
  OR packing is marked Not Required

Required:
If packing is completed and shipment status is set to Packed, Get Rates should become available under the current existing rate prerequisites.

PART C — REOPEN PACKING BEFORE LABEL PURCHASE
==================================================

Required:
Until a label is purchased, the user may reopen packing.

When packing is reopened before label purchase:
- packingStatus should move back to in_progress or equivalent
- shipment should no longer remain operationally Packed
- Get Rates should become blocked again
- selected rate should be cleared or marked stale if package data/packing readiness changes
- audit the reopen action

Important:
Do not let a stale Packed status allow rates after packing is reopened.

Required:
