CURRENT QA SOURCE OF TRUTH — REMAINING FAILURES
==================================================

These are the exact remaining blockers and must be fixed literally:

1) Provider connection status is not syncing correctly
- EasyPost can show success message:
  “Connection to EasyPost verified successfully.”
- But the provider still shows “Not Tested”
- This is incorrect and confusing
- Provider test/health status must update immediately and persist correctly after a successful test

2) Provider prerequisite state is not recalculating correctly
- Even when provider is configured and active, Get Rate can remain disabled
- Tooltip incorrectly says “configure a shipping provider”
- This happens even though provider is already configured
- After repeated attempts it may eventually work, but that is not acceptable
- This is a stale state / bad recalculation / incorrect source-of-truth bug

3) Provider prerequisites do not refresh correctly after shipment edits
- Shipment created without package → Get Rate correctly disabled
- Package added later via Edit → Get Rate still incorrectly disabled
- Tooltip still incorrectly says “configure a shipping provider”
- This means shipment prerequisite state is not recomputed properly after edits

4) Origin and Destination are not editable when they should be
- Before shipment is Dispatched, operators need to be able to correct Origin and Destination if there is an error
- Make Origin and Destination editable in the allowed pre-dispatch edit flow
- If these fields already belong in Edit Shipment, make sure they are actually editable there

5) Need controlled rollback from Dispatched back one step only before carrier scan/acceptance
- If shipment was marked Dispatched but not actually dropped off / needs repackaging / operational exception occurred, there should be a way to go one step back
- This rollback is allowed only if there is NO carrier scan / NO provider acceptance / NO received-by-carrier event
- Once tracking history shows carrier scanned/received/accepted the shipment, rollback must be blocked
- Rejected by Carrier logic should remain exactly as currently designed
