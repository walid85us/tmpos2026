PART B — FINAL ACCEPTED SHIPPING LIFECYCLE RULES
==================================================

Document these final accepted rules clearly:

1) Packing before rates:
- provider-backed Get Rates requires packing completed / Packed or packing Not Required
- package weight and dimensions must be complete where required
- package verification must be current

2) Packed status:
- completing packing marks shipment status as Packed
- Packed means rate/label readiness, not dispatch readiness
- packing may be reopened before label purchase
- reopening packing clears or invalidates selected rate where applicable

3) Label purchase:
- label purchase requires valid provider readiness, selected rate, package readiness, and guardrail clearance where applicable
- after label purchase, packing and package details are locked

4) Dispatch:
- provider shipment cannot be marked Dispatched before label purchase
- manual shipment dispatch requires valid manual handoff requirements
- Dispatched remains distinct from In Transit

5) Tracking:
- In Transit and Delivered must be based on truthful tracking/provider/lifecycle data
- Dispatched must not be counted as In Transit

6) Pickup:
- provider-confirmed pickup must remain distinct from local-only pickup intent
- confirmed pickup dependency must prevent inconsistent shipment cancellation
- pickup orphan / local-dismiss recovery behavior remains accepted

7) Returns:
- reverse logistics flow remains part of Shipping Center
- return shipments must preserve truthful lifecycle and provider-state logic

PART D — FINAL ACCEPTED AUTOMATION RULES
==================================================

Document final accepted automation behavior:

- purpose-driven builder:
  - Flag / Note
  - Queue for Batch
  - Require Review
  - Require Approval
  - Block Action

- Queue for Batch includes Packing Completed
- Require Approval and Block Action use real pre-action triggers only:
  - carrier label about to be purchased
  - pickup about to be requested
  - shipment about to be dispatched
  - batch label purchase about to run

- pre-action hooks must stop the risky action before completion
- Choose Another Rate applies only to label-purchase guardrails
- requester sees Pending Review / Pending Approval
- approver sees Review Request / Approval Request / Review Approval where appropriate
- duplicate approval requests are blocked
- approval clears loop for approved context
- requester note is visible to approver
- dismiss/cancel request requires Request Note
- execution history remains truthful and clickable
- automation outcomes and normal packing history must remain separate
