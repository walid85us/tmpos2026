==================================================

When System Owner grants a Paid Override or paid add-on:

==================================================

If activation mode is Activate after payment:
- feature remains Pending Payment until invoice paid

If activation mode is Activate immediately:

- invoice remains open/due

or
- “Active now — one-time invoice remains open”

==================================================
PART F — PAYMENT STATUS / ENTITLEMENT ACTIVATION
==================================================

Required states:

1) Pending Payment
Used only when:
- invoice exists
- activation mode = Activate after payment
- invoice status = open/overdue
- feature is not active yet

Actions:
- Mark Paid if System Owner authorized
- Cancel Invoice
- no duplicate Trial / Paid Override / Re-trial / Re-grant buttons

2) Paid Override Active / Paid Add-on Active
Used when:
- invoice is paid, or
- activation mode = Activate immediately

Actions:
- Edit Price if supported
- Revoke
- View Invoice

3) Invoice Open + Active
Used when:
- activation mode = Activate immediately
- invoice status = open/overdue
- feature active
- Billing tab still shows invoice as unpaid/open

UI should clearly show:
- Paid Override
- Invoice Open / Payment Due
- feature is active now

4) Cancelled Invoice
If invoice is cancelled before activation:
- feature returns to Not in Plan / Add-on Available
- do NOT show Revoked

If invoice is cancelled after immediate activation:
- either revoke entitlement or require explicit revoke decision
- do not leave inconsistent active feature with cancelled invoice unless explicitly documented

Recommended:
- cancelling an open invoice for immediately activated feature should ask:
  “Cancel invoice and revoke feature access?”
- if confirmed, set entitlement revoked or inactive with audit

5) Revoked
Only after active paid entitlement is revoked.
Do not use Revoked for cancelled modal or cancelled unpaid invoice before activation.
