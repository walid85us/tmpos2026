OVERVIEW CONTENT REQUIREMENTS
=============================

The Overview tab must show these sections:

1. Manual Readiness Summary Cards

Cards:

* DNS Readiness
* SSL Readiness
* Email DNS Readiness
* Security Readiness

Each card must show:

* status
* short explanation
* next action
* source/truth label:

  * Recorded manually
  * Intended state
  * Future integration
  * Not applicable
  * Current app/session data

No live telemetry.

2. Operator Checklist

Show a practical checklist derived from DomainControlPanelOverview:

Examples:

* Domain record created
* Intended DNS records generated
* Records copied to DNS provider
* Manual propagation checked
* SSL readiness recorded
* Email DNS records reviewed
* Security posture reviewed

Checklist must be manual/readiness-based.
Do not imply automatic completion from external systems.

3. Risk Assessment

Show:

* risk level
* risk reasons
* next recommended action

Examples:

* Missing DMARC policy
* Pending manual SSL validation
* DNS records need manual review
* Legacy domain not actively managed
* Registrar security placeholders are future

4. Related Domain Relationships

For root domains:

* list related subdomains.
* show count.
* clicking a related subdomain should open that domain’s overview.

For subdomains:

* show parent root.
* clicking parent root should open parent overview.
* show inherited context.

For platform domains:

* show platform apex relationship or platform-managed context.

5. Recent Activity Summary

Show recent domain-related audit/history entries using DomainAuditSummary where available.

Use language:

* Audit record
* Operator manually marked verified
* Domain status changed
* SSL status changed
* Domain created
* Domain disabled/re-enabled

Do not say immutable unless immutable storage exists.
