Phase 1.2F Strategic Replacement — Replace “Domains” with “Tenant Web Address”

Project:
Multi-Tenant Repair / POS SaaS in Replit with Firebase/Firestore

Current status:

* Phase 1.2F Milestone 0 — Domain Object Model Foundation — ACCEPTED
* Phase 1.2F Milestone 1 — Domain Portfolio Dashboard + Portfolio Table — ACCEPTED
* Phase 1.2F Milestone 2 — Root/Subdomain Overview Workspace — ACCEPTED
* Phase 1.2 / 1.2E remain PASS / HOLD
* Phase 1.3 has NOT started

Strategic product decision:
The previous “Domains” direction is no longer the right product shape.

This SaaS is a repair/POS platform. It is not a DNS host, registrar, SSL certificate platform, or domain-management provider.

Each tenant should primarily receive a platform-managed web address, for example:

tenant.repairplatform.com

That tenant web address becomes the tenant’s operational/customer-facing base URL for:

* main app access
* customer portal
* booking appointments
* repair tracking
* invoice/payment links
* mail-in repair intake if enabled
* future public storefront/landing experience

If the tenant owns their own website or root domain, they can redirect/forward it externally to the platform web address. Full custom domain hosting, DNS verification, SSL automation, registrar integration, DNSSEC, transfer lock, and provider APIs are future/support-assisted capabilities.

Therefore:
Replace the user-facing “Domains” module with a clear **Tenant Web Address** module.
