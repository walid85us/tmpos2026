==================================================
PART A — COMMAND CENTER “MISSION CONTROL” LAYOUT
==================================================

Upgrade Command Center visual hierarchy.

Required sections:

1) Mission Control Header

Add a strong top header with:
- current overall platform state
- last refreshed timestamp
- refresh button
- selected time range
- selected focus mode
- short operational summary

Overall platform state can be derived as:
- Healthy
- Watch
- At Risk
- Critical

Use deterministic rules from support cases, SLA status, high-risk audit events, domain issues, and escalated cases.

Example:
- Critical if any critical support case is overdue or any critical high-risk audit exists.
- At Risk if overdue support cases or failed domains exist.
- Watch if at-risk SLA or warning audit events exist.
- Healthy if no active risk signals exist.

Do not fake infrastructure uptime.

Display truth label:
“Operational state is derived from support, audit, domain, and platform workflow signals — not infrastructure uptime monitoring.”

2) Time Range Control

Add time range filter:
- Today
- 7 days
- 30 days
- All

This should affect:
- audit counts
- high-risk event stream
- recent activity
- tenant risk if practical
- workload charts if practical

If some widgets are all-time only, label clearly.

3) Focus Mode Control

Add operations focus modes:
- Normal
- Watch
- Incident Review

Behavior:
- Normal: show all operational signals
- Watch: prioritize medium/high/at-risk items
- Incident Review: show critical, overdue, escalated, failed, high-risk only

This is a UI filter/state only.
Do not create real incident records in this pass.

4) Operational Pulse Strip

Add a compact row of interactive pulse metrics:
- Open Cases
- Overdue SLA
- Escalated Cases
- High-Risk Audits
- Domain Issues
- Tenants At Risk
- Pending Actions

Each metric should:
- show count
- show trend if derivable, otherwise omit trend
- be clickable or filter the Command Center when practical
- show tooltip/microcopy explaining derivation

PART C — NEXT BEST ACTIONS PANEL
==================================================

Add a deterministic “Next Best Actions” panel.

This is NOT AI.
This is rule-based prioritization.

Examples:
- Review overdue support case
- Investigate critical audit event
- Escalate high-priority support case nearing SLA breach
- Verify failed/pending domain
- Review tenant at risk
- Check add-on readiness blocker
- Review unresolved security note

Each action should show:
- priority
- reason
- related tenant/entity
- recommended next step
- button/link

Rules:
- Critical overdue support cases come first
- Escalated cases come next
- Critical/high-risk audit events next
- Failed domains next
- At-risk SLAs next
- Tenant risk watch items next
- Add-on readiness blockers next

Do not create automated actions.
The panel recommends navigation/actions only.
